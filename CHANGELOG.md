# Changelog

This project follows a lightweight form of [Keep a Changelog](https://keepachangelog.com/)
and uses semantic versioning once tagged releases begin.

## [Unreleased]

## [0.1.1] - 2026-07-06

### Added

- Self-narrating UI anti-pattern guidance.
- Agent-context vs user-action copy separation rules.
- Primary-language guidance for UI labels, states, actions, and protocol/API terms.
- Eval coverage for GPT/Codex self-narrating tool-console copy.

### Changed

- Frontend governance templates now require agent-only context to stay out of primary UI.
- Validator now checks the new UI-copy maturity markers.

## [0.1.0] - 2026-07-06

### Added

- Initial `ui-ux` Codex skill bundle.
- Project-cognition gate, frontend thinking gate, project-stage modes, rescue mode, anti-pattern checks, and frontend governance templates.
- Chinese README and a more complete project landing page.
- Root open-source project documentation and governance files.
- GitHub Actions validation workflow.
- Version file and release packaging script.
- Roadmap and benchmark report template.
- Functional eval set for major `ui-ux` task modes.
- Trigger-boundary eval set with positive and near-miss negative prompts.
- Structural validator with skill, template, script, eval, and open-source
  hygiene checks.

### Changed

- Clarified `ui-ux` as the UI/UX quality gate and governance layer, not a
  replacement for visual implementation skills.
- Expanded frontend governance templates with UX flow, recovery, permission,
  success feedback, and mobile-priority markers.
